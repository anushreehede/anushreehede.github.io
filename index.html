[Master's Thesis] A Data-Driven Error Analysis of Toxicity Detection Models

Prior work in model bias mitigation suggests the use of templates to determine the cause of errors in toxicity models, but our research studies indicate that these results were highly dependent on the templates selected. We focussed our attention to the errors of Jigsaw's Perspective API , a popular, publicly-available tool that scores toxicity of text presented to it. We used the Unintended Bias in Toxicity Classification dataset, and designed a data-driven method to find words that triggered erroneous toxicity scores. We also chose to narrow down our focus to words related to persons, groups and identities since unintended biases in the model can be perpetuated against already maginalized communities; and the engineers of such tools have a responsibility to ensure that their AI doesn't reflect the worst of humanity. In order to provide better context to comments containing such identities, we fine-tuned a BERT-based toxicity model that maintains high performance on the toxicity detection task, and at the same time reduces the error on comments containing person identity/group words from the error trigger word list.

Advised by: Dr. Ani Nenkova and Dr. Byron C. Wallace

PennSearch - Mini Search Engine with Distributed Components

This was a course project where we studied how distributed systems were built and deployed over the Internet. Our goal was to develop a small search engine with the following different components (a) a Mercator-style crawler built on a simple version of Apache Storm, which crawled 200k documents from a variety of different domains, (b) a distributed indexer and PageRanker built using AWS EMR (Hadoop), and (c) a web service for the search engine hosted on AWS EC2, which when given a query, used the stored TFIDF similarities and PageRank to rank all relevant documents and displayed them to the user. All the crawled pages were stored in AWS S3 and the computed TFIDF and PageRank values were stored in AWS RDS (SQL-database).

Course Project of "Internet and Web Systems" at UPenn taught by Dr. Zachary Ives

PennCloud - Distributed Cloud platform with Mail (SMTP) and File Storage Services

This was a course project where we studied the fundamental concepts behind distributed systems, and the design principles for building large-scale computational systems. Our goal was to develop a cloud platform in C++ that supports user account creation, an email service compliant with SMTP and a hierarchical file storage service similar to Google Drive. All data storage was done using a key-value store in backend (similar to Google's BigTable) with simple operations such as GET, PUT and CPUT. Communication in the distributed network was carried out using Protobuf over TCP. We built the backend servers which supported sequential consistency, primary-based replication, fault tolerance and recovery, and the frontend servers which supported HTTP requests, cookies, load balancing and fault tolerance.

Course Project of "Software Systems" at UPenn taught by Dr. Linh Thi Xuan Phan

Explainability for Multiple-Choice Science Question-Answering

AristoRoBERTa is the state-of-the-art choice for building multiple-choice QA models developed by Allen AI . As input, it takes in the question, answer option, and a set of ten supporting context sentences for each answer option. It then predicts the correct answer choice for the question. We define two types of probing mechanisms (a) single (where only one out of ten context sentences is used for scoring) and (b) leave-one-out (where all but one out of ten contect sentences are used for scoring), and probe the trained model itself to rank the context sentences in order of their importance toward answering the question. We thus obtained explanations out of black-box models. Further, a comparable performance is obtained when only a subset of most important context sentences are provided to the model to make an answer prediction.

Advised by: Dr. Dan Roth

Visual Question-Answering: Using Object Information as Metadata

One of the most popular datasets for visual question-answering is the Visual QA Dataset. In this course project, we were to build three models: (a) non-deep learning baseline: concatenated a bag-of-words from the question, and ResNet-50 features from the image, (b) deep learning baseline: swapped above with an embedding layer and GRU, and (c) and advanced deep learning model: the deep learning baseline with attention modules, and a Faster-rCNN feature module for object positions in the image. The advanced model was purely experimental and interestingly gave a 5% increase in accuracy compared to the baselines. The models were trained and tested on AWS Sagemaker.

Course Project of "Deep Learning for Data Science" at UPenn taught by Dr. Konrad Kording

Predicting Success of Kickstarter Projects

Kickstarter is an online platform, home to the popular method of crowdfunding, where several people contribute small amounts of money to a new startup or project. Typically, these projects define a target budget and some additional information such as country, project description, deadline, backers, etc. If the project is able to raise funds to meet it's target amount, then it is declared as a success on Kickstarter. In this project, we applied a variety of classical machine learning techniques to determine whether a given Kickstarter project will be successful or not. We collected some of our data from the and scraped the remaining from the Kickstarter website. We also performed a detailed feature selection process and visualized the data extensively. We experimented with a range of classification and regression approaches for the problem, and determined that gradient boosting gave the best results for the task.

Course Project of "Machine Learning" at UPenn taught by Dr. Lyle Ungar
